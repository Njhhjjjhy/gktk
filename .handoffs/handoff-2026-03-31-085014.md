## Architecture and design notes

### Page structure (4 sections)

```
navigation  — fixed top, mix-blend-mode: difference, icon left + hamburger right
hero        — 100vh, light bg (--bg), title + subtitle + 3-col concept grid
tilt-cards  — dark bg (--bg-dark), 7 cards (was 8, concept moved to hero)
footer      — dark bg, placeholder copy, icon brand
```

### Navigation simplification

The navigation was reduced from a full 3-column grid (brand | links | menu button) to a simple flex row (icon | menu button). The nav links were completely removed — all navigation now goes through the fullscreen menu overlay. The menu button is visible at ALL breakpoints (previously was mobile-only). This is a design decision: the site is a presentation deck, not a multi-page site, so persistent nav links are unnecessary.

### Hero becomes light-themed

The hero switched from dark (`--bg-dark` background, `--fg-invert` text) to light (`--bg` background, `--neutral-950` text). This creates visual contrast between the hero and the dark tilt-cards-section below. The hero mask parts were also updated to `--bg` to match.

### Concept content promoted to hero

Card 1 (Concept — TiltCardA with cycling background) was removed from the tilt-cards-section. Its content (3-column grid: Furnished housing, Localized support, Professional management) was moved into `hero__bottom` as a new `.hero__grid` element. This makes the concept information immediately visible without scrolling, like the Radiance reference site's hero approach.

### Page transition wipe

The `.page-transition-wipe` element (fixed overlay, `translateY(100%)`) is now wired to anchor navigation clicks from the menu overlay. The sequence:
1. Wipe in: `y: 0` over 0.8s with `power3.inOut`
2. Instant scroll to target (Lenis `immediate: true`)
3. Wipe out: `y: "-100%"` over 0.8s with `power3.inOut`
4. Reset: `y: "100%"` via GSAP `.set()`

Falls back to smooth Lenis scroll or native `scrollIntoView` if the wipe element is missing.

### Dead code policy

The user explicitly decided to keep some unused code for future use:
- `.tilt-card__grid-image` — CSS ready for grid items with images
- `--radius-s`, `--radius-m` — design tokens for future components

Everything else unused was removed. The easing custom properties were removed because GSAP handles easing via string names directly — CSS custom properties for easings were never consumed.
