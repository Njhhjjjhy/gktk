## Session goal

Build a QA breakpoint preview tool for responsive layout testing. The tool renders the current site inside a resizable iframe overlay, with device presets and CSS breakpoint pills, triggered by a floating button or keyboard shortcut. Dev-only (localhost), zero dependencies, self-contained IIFE in a single JS file. The user provided a detailed spec as their prompt.

### Phase 3: Building the tool — `public/js/qa-breakpoints.js`

Created the full IIFE. Key implementation details:

- **Recursive iframe prevention**: Checks `window.location.search.indexOf("_qa=1")` and exits early if present. The iframe loads the current page with `?_qa=1` appended (handles both `?` and `&` cases).
- **Localhost guard**: Also exits if not localhost/127.0.0.1/file: protocol (double guard — both the loader component and the script itself check).
- **No ES6+ syntax**: Used `var` throughout, `function` declarations, `.indexOf()` instead of `.includes()`, `textContent` array `.join()` instead of template literals. Per the spec constraint for older tooling compatibility.
- **9 device presets**: iPhone 17 Pro (393x852, default), iPhone SE, iPad mini, iPad Air, iPad Pro 11", iPad Pro 13", Laptop 13", Desktop, Desktop LG.
- **2 breakpoint pills**: `1100px max` and `767px max`, highlighted with teal accent when active at current iframe width.
- **Toolbar**: Dark bar (#1e2027) with device buttons, breakpoint pills, width/height number inputs, Apply button, rotate button, dimension label, close button. All separated by vertical dividers.
- **Viewport**: Dark background (#111216) with CSS dot grid pattern (`radial-gradient` at 24px intervals), iframe centered with border, rounded corners, white background, drop shadow, smooth CSS transitions on width/height changes (0.3s cubic-bezier).
- **Keyboard shortcut**: `Cmd+Shift+Q` on Mac, `Ctrl+Shift+Q` on others. Mac detected via `navigator.platform.indexOf("Mac")`. Uses `e.metaKey` on Mac (NOT `e.ctrlKey` — spec was explicit about this).
- **Floating trigger**: Fixed bottom-right (16px inset), z-index 199999, teal circle (#0F6466), white "QA" text, tooltip shows shortcut. Hover scales up slightly.

## Architecture and design notes

### Iframe isolation pattern
The tool uses a `?_qa=1` URL parameter as a recursion guard. The main page loads the script, the script creates an iframe pointing to the same URL + `?_qa=1`, and the script exits early when it detects that parameter. This means the iframe gets a clean copy of the page without the QA overlay.

### Localhost-only loading (two layers)
1. **Component layer**: `QALoader.tsx` checks `window.location.hostname` in a `useEffect` and only renders the `<Script>` tag on localhost.
2. **Script layer**: `qa-breakpoints.js` itself also checks hostname at the top of the IIFE and exits early if not localhost. Belt and suspenders.

### ES5 constraint
The JS file uses only ES5 syntax (var, function expressions, .indexOf, .join) despite the rest of the project being TypeScript/React. This was a spec requirement for older tooling compatibility. The `QALoader.tsx` wrapper is modern TypeScript since it runs through the Next.js build pipeline.

### Style injection
All CSS is injected via a single `<style>` element created in JS, using an array of CSS strings joined with newlines. No external stylesheet dependency. All selectors are prefixed with `#qa-` or `.qa-` to avoid collisions with the host page.
