## Architecture and design notes

- **App Router pattern:** The project uses Next.js App Router (not Pages Router). All routes live under `src/app/`. Layouts nest automatically. Server Components are the default.
- **Styling approach:** Tailwind CSS 4 with PostCSS plugin. No `tailwind.config.js` file - Tailwind 4 uses CSS-native configuration via `@theme inline` blocks in `globals.css`.
- **Font strategy:** Geist Sans and Geist Mono loaded via `next/font/google` with CSS variable injection (`--font-geist-sans`, `--font-geist-mono`). Applied on the `<html>` element.
- **Dark mode:** Implemented via `prefers-color-scheme: dark` media query setting CSS custom properties. Not using Tailwind's `dark:` class-based dark mode toggle - it's system-preference-only via the CSS in `globals.css`, though `dark:` utility classes are also used in `page.tsx`.
- **React Compiler:** Enabled globally in `next.config.ts`. This means manual `useMemo`, `useCallback`, and `React.memo` are unnecessary - the compiler handles memoization automatically.
