## Problems encountered and how they were solved

### 1. Whether @tailwindcss/postcss interferes with custom CSS
- **Symptom:** Uncertainty about whether removing `@import "tailwindcss"` from globals.css would cause the PostCSS plugin to strip or mangle custom CSS.
- **Root cause:** Non-issue. The plugin only processes Tailwind-specific directives. Custom CSS passes through untouched.
- **Resolution:** Confirmed via Explore agent that checked compiled CSS in `.next/` directory — all custom classes and properties preserved correctly.
- **Future note:** Safe to write pure custom CSS with `@tailwindcss/postcss` in the PostCSS config. It's a no-op for non-Tailwind CSS.

### 2. text-mask__inner elements invisible without JS
- **Symptom:** All text using the `.text-mask__inner` pattern starts at `transform: translateY(105%)` which hides it. If GSAP doesn't load or animations don't fire, ALL text is invisible.
- **Root cause:** The animation pattern requires GSAP to reveal text. No CSS fallback.
- **Resolution:** Not yet fixed. GSAP loads correctly in the current implementation, so text reveals work. But this is fragile — a JS error could hide all content.
- **Future note:** Consider adding a CSS fallback like `.js-loaded .text-mask__inner { transform: translateY(105%); }` and only applying the hidden state after JS confirms loaded.

### 3. Spec's section-spacing values were extreme
- **Symptom:** `--section-spacing-desktop: 26rem` (416px) padding on each side created 832px of empty space between content areas.
- **Root cause:** The spec values were literally from the reference sites but didn't account for the combined effect when used as padding-top AND padding-bottom.
- **Resolution:** First replaced with `clamp(5rem, 10vw, 10rem)`, then in the final overhaul switched to `min-height: 100vh` sections with `align-items: center` — content naturally centers in the viewport.

### 4. Nav legibility over dark/light sections
- **Symptom:** First implementation used glassmorphism with light background — unreadable over dark hero.
- **Root cause:** Glassmorphism assumes light page background. The dark hero broke this.
- **Resolution:** First tried transparent → glassmorphism transition on scroll. Final solution: `mix-blend-mode: difference` as specified in the agency portfolio reference — auto-inverts text color against any background.

## Learnings and discoveries

1. **vw-based body font is the key to the agency portfolio aesthetic.** Setting `body { font-size: 0.833vw }` and using `em` throughout makes everything scale proportionally. This is fundamentally different from rem/px approaches and is what gives the reference site its "designed for the screen" feel. At 1440px viewport, 1em = 12px. At 1920px, 1em = 16px.

2. **mix-blend-mode: difference** on fixed nav is the industry-standard pattern for full-viewport dark/light section marketing sites. No need for scroll-triggered class changes, intersection observers, or separate dark/light nav variants. Combined with `pointer-events: none` on the header (with `all` on children) and white text, it handles all cases.

3. **The agency portfolio card stack** uses `rotationX: 40deg` for entrance (card tilted toward viewer from below) and `rotationX: -40deg` for exit (card tilts away above), creating a deck-peeling effect. The background color transition (dark → accent → dark) adds drama.

4. **Next.js 16's `@tailwindcss/postcss`** plugin is non-destructive to custom CSS. Safe to keep installed while writing pure custom CSS.

5. **Noto Sans JP** is imported via `next/font/google` with explicit weights `["400", "600", "700"]`. The CSS variable `--font-noto-sans-jp` is injected on `<html>`. In the final version, body font-family references it directly: `'Noto Sans JP', sans-serif`.

6. **The user is building a real investment pitch site** targeting Singapore-based family offices (SFOs) for semiconductor housing in Kumamoto, Japan. The content includes real financial data (¥2B total investment, 10.25% IRR, 1.65x MOIC), specific company names (TSMC, Sony, Tokyo Electron, ASML, CapitaLand, MoreHarvest, Chateau Life), and legal structures (TK/Tokumei Kumiai silent partnership). ## Architecture and design notes

### Scaling system
The site uses a **vw-based fluid scaling** approach from the agency portfolio reference. `body { font-size: 0.8333vw }` sets 1em = 1/120 of viewport width. At 1440px, 1em ≈ 12px. All component sizes use `em` units, making everything scale proportionally with the viewport. On mobile (<767px), body font-size locks to `1rem` (16px) to prevent illegibly small text.

### Navigation pattern
`mix-blend-mode: difference` on the fixed header with `pointer-events: none` (children get `pointer-events: all`). All nav text is white — the blend mode inverts it over light sections automatically. No glassmorphism, no scroll-triggered state changes. This is the cleanest approach for full-viewport dark/light alternating sections.

### Section architecture
Every content section uses `min-height: 100vh; display: flex; align-items: center` to fill the viewport with centered content. The product section breaks this pattern with a 500vh scroll-driver wrapper and GSAP-pinned slides.

### Animation architecture
A single client component (`Orchestrator.tsx`) handles all interactivity via imperative DOM manipulation in a `useEffect`. This matches the spec's architecture (GSAP querySelector-based, not React state). The component returns `null` — it's purely a side-effect runner. Cleanup kills all ScrollTriggers and destroys Lenis on unmount.

### Text reveal pattern
`.text-mask` (overflow: hidden) wraps `.text-mask__inner` (translateY: 105%). GSAP animates `yPercent: 0` to reveal. Used on all section headings and the hero title. Each word/line is wrapped in its own mask for staggered reveals.

### Card-stack pattern
5 slides in the same grid cell (grid-area: 1/1). First slide has bg that scales from 0.075 to 1 (dramatic zoom-in). Each subsequent slide enters at rotationX: 40deg (tilted toward viewer) and scales from 1.1→1. On exit, rotationX goes to -40deg with scale 0.7 and y: -6%. The wrapper background color transitions from dark to amber (accent) and back during the scroll, adding drama behind the rounded card edges.

### Dark/light rhythm
Hero (dark) → Concept (light) → Persona (dark) → Product (dark wrapper, card content varies) → IRR (light) → Risk (dark) → Exit (light) → Footer (dark). This creates visual contrast and section delineation without explicit dividers.
